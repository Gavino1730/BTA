# Support

## Getting help

Use GitHub Issues for bugs and feature requests.

Before opening an issue:

1. Check existing issues for duplicates.
2. Include environment, steps to reproduce, expected behavior, and actual behavior.
3. Add logs/screenshots when possible.

## Questions and usage help

For setup and deployment:

- `README.md`
- `HOSTING_SETUP.md`
- `DEPLOYMENT.md`

## Security issues

Do not post security vulnerabilities publicly.

Please report using the process in `SECURITY.md`.
